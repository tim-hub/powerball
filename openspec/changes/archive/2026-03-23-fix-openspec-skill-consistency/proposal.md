## Why

The 8 openspec-rewrite skills generated by the openspec CLI have inconsistent frontmatter, duplicated content, second-person writing style violations, and oversized bodies. This makes skill selection unreliable (trigger phrase collisions) and maintenance harder (duplicated guidelines drift). The skills need to be normalized to match the repo's established conventions from `core/` and `workplace/`.

## What Changes

- Reorder frontmatter on all 8 generated skills to match repo convention: `name` → `description` → `user-invocable` → `model` → others
- Add missing `argument-hint: "[change name]"` to all 8 change-management skills
- Add missing `model` key to 6 skills (apply, continue, new, sync, verify, explore)
- Extract duplicated "Artifact Creation Guidelines" from `openspec-propose` and `openspec-continue-change` into a shared `references/artifact-guidelines.md`
- Move dialogue examples from `openspec-explore` body to `references/explore-examples.md`
- Rewrite second-person sections in `openspec-explore` to imperative form
- Move output template and heuristics from `openspec-verify-change` to `references/`

## Capabilities

### New Capabilities

- `skill-frontmatter-normalization`: Covers reordering frontmatter keys and adding missing keys (`argument-hint`, `model`) across all 8 generated skills
- `skill-body-simplification`: Covers extracting duplicated/verbose content to `references/` directories and rewriting second-person to imperative

### Modified Capabilities

(none — this change modifies skill files directly, no existing specs affected)

## Impact

- 8 SKILL.md files under `openspec-rewrite/skills/` will be edited (frontmatter reorder + missing keys)
- 2 new `references/` directories created (explore, verify-change)
- 1 shared `references/` created for artifact creation guidelines
- `openspec-propose` and `openspec-continue-change` bodies shrink as duplicated section is replaced with a reference pointer
- `openspec-explore` body shrinks by ~100 lines (examples extracted)
- `openspec-verify-change` body shrinks by ~50 lines (templates/heuristics extracted)
