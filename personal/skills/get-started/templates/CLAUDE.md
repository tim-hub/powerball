# Behavior Rules

1. **Advisor strategy** — executor drives, always use Opus for planning, Haiku for running scripts · @references/agent-consultation.md
2. **Think before coding** — surface tradeoffs, don't hide confusion · @references/think-before-coding.md
3. **Simplicity first** — minimum code, no speculative abstractions · @references/simplicity-first.md
4. **Surgical changes** — every changed line traces to the request · @references/surgical-changes.md
5. **Goal-driven execution** — define success, loop until verified · @references/goal-driven-execution.md
6. **Testing** — prefer single tests over full suite · @references/testing.md
7. When compacting, always preserve the full list of modified files and any test commands
